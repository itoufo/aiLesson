# 第5章：AI活用演習

## 学習目標

この章を終えると、以下のことができるようになります：

- AIを使って文章の要約・校正・言い換えができる
- 業務で使えるプロンプトを実践的に作成できる
- AIの出力を適切に評価・修正できる
- 自己調整学習の手法を活用できる

---

## 5.1 要約スキル

### 5.1.1 要約の基本

![要約の3つの種類](/docs/images/01_入門/05_summary_types.jpeg)

### 5.1.2 要約プロンプトテンプレート

```markdown
## 要約依頼

### 要約の種類
[抽出型 / 圧縮型 / 再構成型]

### 目標文字数
[文字数]文字程度

### 重視する観点
- [ ] 結論・主張を最優先
- [ ] 具体的なデータ・数字を含める
- [ ] 時系列を維持
- [ ] 因果関係を明確に

### 出力形式
[箇条書き / 段落形式 / 見出し付き]

### 要約対象
---
[ここに要約したい文章を貼り付け]
---
```

### 5.1.3 演習：ニュース記事の要約

**課題**: 以下の文章を3種類の方法で要約してください。

```
【元の文章】（例）
政府は本日、2024年度の経済成長率見通しを発表した。実質GDP
成長率は前年度比1.3%増と予測され、前回予測の1.0%から上方
修正された。主な要因として、個人消費の回復と設備投資の増加
が挙げられている。一方で、物価上昇率については2.5%と依然
として高い水準が続く見込みで、家計への影響が懸念されている。
政府は「賃上げと物価上昇の好循環を目指す」との方針を示した。
```

**演習タスク**:
1. 50文字以内の一文要約を作成
2. 箇条書き3点の要約を作成
3. ビジネスパーソン向けの要約を作成

## 5.2 校正スキル

### 5.2.1 AIによる校正の種類

| 校正の種類 | 内容 | プロンプト例 |
|------------|------|--------------|
| **誤字脱字** | 漢字・送り仮名の間違い | 「誤字脱字をチェックしてください」 |
| **文法** | 文法的な誤り | 「文法の誤りを指摘してください」 |
| **表現** | 不自然な表現 | 「より自然な表現に修正してください」 |
| **論理** | 論理の矛盾・飛躍 | 「論理の一貫性を確認してください」 |
| **トーン** | 文体の統一 | 「敬語を統一してください」 |

### 5.2.2 校正プロンプトテンプレート

```markdown
以下の文章を校正してください。

## 校正のポイント
- [ ] 誤字脱字
- [ ] 文法の誤り
- [ ] 表現の不自然さ
- [ ] 敬語の統一
- [ ] 論理の一貫性

## 出力形式
1. 修正箇所を指摘（元の表現→修正後）
2. 修正理由を簡潔に説明
3. 最後に修正後の全文を提示

## 校正対象
---
[ここに校正したい文章を貼り付け]
---
```

### 5.2.3 演習：ビジネスメールの校正

**課題**: 以下のメールを校正してください。

```
【校正対象】
件名：明日の会議の件

山田部長

お疲れさまです。
明日の会議ですが、資料の準備が遅れていて、
もうすこしだけ時間が必要です。
会議を午後からに変えていただくことは可能でしょうか。
ご検討いただければ幸です。

田中
```

**チェックポイント**:
- 敬語の使い方は適切か
- 誤字・脱字はないか
- ビジネスメールとして適切な表現か
- 情報は十分か

## 5.3 言い換えスキル

### 5.3.1 言い換えの目的

![言い換えの主な目的](/docs/images/01_入門/05_paraphrase_purposes.jpeg)

### 5.3.2 言い換えプロンプトテンプレート

```markdown
以下の文章を言い換えてください。

## 元の文章の特徴
- 対象読者：[元の対象読者]
- トーン：[元のトーン]
- 目的：[元の目的]

## 言い換え後の条件
- 対象読者：[新しい対象読者]
- トーン：[新しいトーン]
- 目的：[新しい目的]
- 文字数：[目安の文字数]

## 言い換え対象
---
[ここに言い換えたい文章を貼り付け]
---
```

### 5.3.3 演習：読者別言い換え

**課題**: 以下の技術説明を3種類の読者向けに言い換えてください。

```
【元の文章】
クラウドコンピューティングとは、インターネット経由でコンピューティ
ングリソース（サーバー、ストレージ、データベース、ネットワーク、
ソフトウェアなど）をオンデマンドで利用できるサービス形態です。
従来のオンプレミス環境と異なり、初期投資を抑えながら必要に応じて
スケールアップ・ダウンが可能です。
```

**言い換え先**:
1. 小学生向け（わかりやすく、身近な例えを使って）
2. 経営者向け（ビジネスメリットを中心に）
3. キャッチコピー（20文字以内）

## 5.4 複合演習

### 5.4.1 演習：会議議事録の作成

**シナリオ**: 以下の会議メモから議事録を作成してください。

```
【会議メモ】
日時：1/15 10:00-11:30
参加：山田部長、佐藤課長、田中、鈴木
議題：新製品発売スケジュール

・山田部長：3月発売を目指したい
・佐藤課長：製造ラインの準備、2月末に完了予定
・田中：マーケティング施策、SNS広告中心に検討中
・鈴木：パッケージデザイン、2案を次回提示
・予算：500万円で承認
・次回：1/22 同時刻

懸念事項：原材料の納期遅延リスク
→ 佐藤課長がサプライヤーと交渉
```

**タスク**:
1. 正式な議事録形式で作成
2. 決定事項と懸念事項を明確に区別
3. アクションアイテムをリスト化

### 5.4.2 演習：提案書の作成支援

**シナリオ**: 上司から「社内のペーパーレス化」について提案書を作成するよう依頼されました。

**タスク**:
以下のステップでAIを活用して提案書の骨子を作成してください。

```
【ステップ1】アイデア出し
「ペーパーレス化のメリットを5つ挙げてください」

【ステップ2】課題の整理
「ペーパーレス化を進める際の課題と対策を整理してください」

【ステップ3】構成案の作成
「提案書の構成案を作成してください」

【ステップ4】各セクションの執筆
「提案書の背景セクションを200文字で書いてください」
```

### 5.4.3 演習：カスタマーサポート対応

**シナリオ**: 以下の顧客からのクレームメールに対する返信を作成してください。

```
【顧客からのメール】
注文した商品が届いたのですが、箱が潰れていて中の商品にも
傷がついていました。楽しみにしていたのに、とても残念です。
交換か返金をお願いしたいです。
```

**タスク**:
1. まずAIに返信案を作成させる
2. 出力を評価し、改善点を指摘
3. 改善したプロンプトで再度作成
4. 最終的な返信文を完成させる

## 5.5 自己調整学習

### 5.5.1 自己調整学習とは

**自己調整学習**は、自分の学習プロセスを自分でコントロールする学習方法です。

![自己調整学習のサイクル](/docs/images/01_入門/05_self_regulated_learning.jpeg)

### 5.5.2 AIを活用した自己調整学習

```markdown
## AIへの学習相談プロンプト

### 現状の説明
私は[職業/立場]で、[現在のスキルレベル]です。

### 学習目標
[達成したい目標]を[期間]で達成したいと考えています。

### 質問
1. この目標は現実的ですか？
2. どのような学習ステップを踏むべきですか？
3. つまずきやすいポイントは何ですか？
4. おすすめの学習リソースはありますか？
```

### 5.5.3 学習の振り返りテンプレート

```markdown
## 今日の学習振り返り

### 学習内容
- 学習したこと：[内容]
- 所要時間：[時間]

### 理解度チェック
以下の項目を5段階で自己評価してください：

| 項目 | 評価（1-5） |
|------|-------------|
| 概念の理解 | |
| 実践での応用力 | |
| 説明できるレベル | |

### 振り返り質問
1. 今日学んだことで最も重要なポイントは？
2. まだ理解が不十分な点は？
3. 明日の学習で優先すべきことは？

### AIへの確認
「私の理解が正しいか確認してください：[自分の理解を説明]」
```

## 5.6 演習の振り返りと評価

### 5.6.1 自己評価チェックリスト

```
【要約スキル】
□ 目的に応じた要約の種類を選択できる
□ 指定された文字数で要約できる
□ 重要なポイントを漏らさず含められる

【校正スキル】
□ 誤字脱字を発見できる
□ 文法の誤りを指摘できる
□ 表現の改善提案ができる

【言い換えスキル】
□ 読者に合わせた言い換えができる
□ トーンの調整ができる
□ 目的に応じた文体変換ができる

【プロンプト作成】
□ 目的を明確に伝えられる
□ 具体的な条件を設定できる
□ 出力を評価し改善できる
```

### 5.6.2 スキルレベル判定

| レベル | 基準 |
|--------|------|
| ★☆☆☆☆ | プロンプトを書けるが、期待通りの出力を得られないことが多い |
| ★★☆☆☆ | 基本的なプロンプトで期待通りの出力を得られる |
| ★★★☆☆ | 複雑なタスクでも適切なプロンプトを設計できる |
| ★★★★☆ | 出力を評価し、プロンプトを改善するサイクルを回せる |
| ★★★★★ | 業務で自在にAIを活用し、他者に指導もできる |

## 章末まとめ
