## 学習目標

この章を終えると、以下のことができるようになります：

- AI活用ポートフォリオの意義と構成を理解できる

## 3.3 プロジェクト紹介の書き方

### 3.3.1 プロジェクト紹介テンプレート

```markdown
# プロジェクト名：[タイトル]

## 概要
- **期間**：[開始日] 〜 [終了日]
- **チーム**：[人数]名
- **役割**：[自分の役割]
- **使用ツール**：[使用したAIツール等]

## 背景と課題
[プロジェクトの背景、取り組んだ課題を説明]

## 目標
[プロジェクトで達成しようとしたこと]

## アプローチ（P-A-I-Cの記録）

### Prompt（問いかけ）
[最初にどのような問いを立てたか]

### Ask（質問）
[AIとのやりとりで深掘りしたポイント]

### Iterate（反復）
[どのように改善を繰り返したか]
- 第1回フィードバック：[内容]
- 第2回フィードバック：[内容]
- ...

### Create（創造）
[最終的に何を作り上げたか]

## 成果物
[成果物の概要、可能であれば画像やリンク]

## 成果・効果
- 定量的成果：[数値で表せる成果]
- 定性的成果：[数値以外の成果]
- ステークホルダーからのフィードバック：[あれば]

## 振り返り

### 学んだこと
- [学び1]
- [学び2]
- [学び3]

### 苦労した点と克服方法
- [課題]：[克服方法]

### 次に活かしたいこと
- [今後の改善点・発展]

## AI活用のポイント
- 効果的だった活用方法：[具体例]
- うまくいかなかった活用方法：[具体例と学び]
- AI活用で実現できたこと：[成果]
```

### 3.3.2 良いプロジェクト紹介の例

```markdown
# プロジェクト名：地域観光PR動画企画

## 概要
- **期間**：2024年10月1日 〜 10月31日（4週間）
- **チーム**：4名（リサーチ、企画、ライティング、PM）
- **役割**：企画・分析担当
- **使用ツール**：Claude、ChatGPT、Notion AI

## 背景と課題
A町（人口2万人）の観光客が5年連続で減少。
町役場から「若年層向けのPR施策」の企画を依頼された。
課題は、町の魅力が若者に届いていないこと。

## 目標
1. 20-30代に響くPRコンセプトの策定
2. SNS動画3本の企画・構成案作成
3. 1年間のPRロードマップ策定

## アプローチ（P-A-I-C）

### Prompt
「A町の観光資源を洗い出し、20-30代に響く
切り口を10個提案してください」からスタート。

### Ask
- 「○○という切り口について、具体的な表現方法は？」
- 「競合（B市、C村）との差別化ポイントは？」
- 「インスタ映えする具体的なスポット提案は？」

### Iterate
計8回の反復を実施。
- 1-3回目：コンセプトの方向性を模索
- 4-5回目：具体的なコンテンツ案を精緻化
- 6-8回目：ロードマップの実現可能性を検証

### Create
最終的に「#知られざるA町」をコンセプトに、
地元民だけが知る穴場スポットを紹介する
動画企画を完成。

## 成果物
1. PRコンセプト提案書（15ページ）
2. SNS動画企画書×3本（絵コンテ付き）
3. 年間PRロードマップ
4. 効果測定KPI設計書

## 成果・効果
- 町役場から「採用」の評価を獲得
- 実際に動画1本が制作・公開（視聴回数2万回）
- 町のSNSフォロワー15%増加に貢献

## 振り返り

### 学んだこと
- AIは発散的なアイデア出しに非常に有効
- ローカルな情報は人間が補完する必要がある
- 反復の質が最終成果物の質を決める

### 苦労した点と克服方法
- AIが一般的な観光施策に偏りがち
  → 地元の人へのヒアリング結果を入力して軌道修正
- チーム間の認識ズレ
  → 週次で成果物を共有し、早めにすり合わせ

### 次に活かしたいこと
- 最初のPromptにもっと具体的な制約を入れる
- ローカル情報のインプットを早い段階で行う

## AI活用のポイント
- **効果的**：競合分析、ペルソナ設定、コンセプト案出し
- **補完が必要**：地元ならではの情報、最新トレンド
- **AI活用で実現**：通常2ヶ月かかる企画を1ヶ月で完了
```

## 3.4 ポートフォリオの作成形式

### 3.4.1 形式の選択肢

| 形式 | メリット | デメリット | 向いている人 |
|------|----------|------------|--------------|
| **Webサイト** | 公開しやすい、更新容易 | 作成スキルが必要 | IT系、公開したい人 |
| **Notion** | 構造化しやすい、共有容易 | デザインの自由度低 | 日常使いしている人 |
| **PDF** | どこでも見られる、印刷可 | 更新が手間 | フォーマル用途 |
| **スライド** | プレゼンしやすい | 詳細記載が難しい | 発表機会がある人 |

### 3.4.2 Notionでのポートフォリオ構成例

```
📁 AI活用ポートフォリオ
├── 📄 トップページ（自己紹介）
├── 📊 スキルマップ（データベース）
├── 📁 プロジェクト
│   ├── 📄 プロジェクト1：[タイトル]
│   ├── 📄 プロジェクト2：[タイトル]
│   └── 📄 プロジェクト3：[タイトル]
├── 📝 学習の記録
│   ├── 📄 第1段階（入門）振り返り
│   ├── 📄 第2段階（応用）振り返り
│   └── 📄 第3段階（発展）振り返り
└── 🏅 認定・バッジ
```

## 3.6 実践演習

### 演習：ポートフォリオの作成

```
【課題】
これまで取り組んだプロジェクト（実際または演習）を
ポートフォリオとしてまとめてください。

作成するもの：
1. 自己紹介・プロフィール
2. スキルマップ
3. プロジェクト紹介（最低1つ）
4. 学習の振り返り

形式：
- Notion、PDF、スライドのいずれか

評価基準：
- 内容の充実度
- 構成のわかりやすさ

- 振り返りの深さ
```
