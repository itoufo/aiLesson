## 2.2 情報の分類と構造化

### 2.2.1 MECE（ミーシー）の原則

**MECE**（Mutually Exclusive, Collectively Exhaustive）は「漏れなく、重複なく」という分類の原則です。

![MECEの概念](/docs/images/02_応用/02_mece_concept.jpeg)

### 2.2.2 MECEでの情報整理プロンプト

```markdown
以下の情報をMECEの原則に従って整理してください。

## 分類の条件
- 各カテゴリは相互に排他的（重複なし）
- 全体で漏れがない（網羅的）
- 分類の軸を明確に

## 整理対象
[整理したい情報を記載]

## 出力形式
分類軸：[軸の説明]

カテゴリ1：[名称]
- 定義：[このカテゴリに含まれるものの定義]
- 該当項目：[項目リスト]

カテゴリ2：...
```

### 2.2.3 ピラミッド構造

情報を階層的に整理する方法です。

![ピラミッド構造](/docs/images/02_応用/02_pyramid_structure.jpeg)

### 2.2.4 ピラミッド構造での整理プロンプト

```markdown
以下の情報をピラミッド構造で整理してください。

## 情報
[整理したい情報・データ・事実を記載]

## 整理の目的
[この情報を使って何を伝えたいか]

## 出力形式

### 結論（メインメッセージ）
[1文で表現]

### 根拠1：[サブメッセージ]
- 事実A：[データ・事例]
- 事実B：[データ・事例]

### 根拠2：[サブメッセージ]
- 事実A：[データ・事例]
- 事実B：[データ・事例]

### 根拠3：[サブメッセージ]
...
```

## 2.3 ビジネスフレームワークの活用

### 2.3.1 主要フレームワーク一覧

| フレームワーク | 用途 | 構成要素 |
|----------------|------|----------|
| **SWOT分析** | 戦略立案 | 強み・弱み・機会・脅威 |
| **3C分析** | 市場分析 | 顧客・競合・自社 |
| **4P分析** | マーケティング | 製品・価格・流通・販促 |
| **5W1H** | 情報整理 | Who・What・When・Where・Why・How |
| **PEST分析** | 環境分析 | 政治・経済・社会・技術 |
| **バリューチェーン** | 事業分析 | 主活動・支援活動 |

### 2.3.2 SWOT分析プロンプト

```markdown
以下の情報を基に、SWOT分析を行ってください。

## 対象
[分析対象（企業名、事業名、プロジェクト名など）]

## 背景情報
[関連する情報を記載]

## 出力形式

### SWOT分析結果

| | プラス要因 | マイナス要因 |
|---|---|---|
| **内部環境** | **強み (Strengths)** | **弱み (Weaknesses)** |
| | ・[項目1] | ・[項目1] |
| | ・[項目2] | ・[項目2] |
| **外部環境** | **機会 (Opportunities)** | **脅威 (Threats)** |
| | ・[項目1] | ・[項目1] |
| | ・[項目2] | ・[項目2] |

### クロスSWOT分析
- SO戦略（強み×機会）：[戦略案]
- WO戦略（弱み×機会）：[戦略案]
- ST戦略（強み×脅威）：[戦略案]
- WT戦略（弱み×脅威）：[戦略案]
```

### 2.3.3 5W1H整理プロンプト

```markdown
以下の内容を5W1Hで整理してください。

## 整理対象
[整理したい内容・状況]

## 出力形式

| 要素 | 質問 | 回答 |
|------|------|------|
| Who | 誰が？ | |
| What | 何を？ | |
| When | いつ？ | |
| Where | どこで？ | |
| Why | なぜ？ | |
| How | どのように？ | |

### 整理結果のサマリー
[5W1Hを統合した説明文]
```

## 2.5 ドキュメント作成の実践

### 2.5.1 報告書テンプレート

```markdown
以下の情報を基に、報告書を作成してください。

## 報告の目的
[この報告で何を伝えたいか]

## 読者
[誰が読むか、その人の関心事]

## 元になる情報
[報告に含めたい情報・データ]

## 出力形式

# [報告書タイトル]

## エグゼクティブサマリー
[全体を3-5行で要約]

## 1. 背景・目的
[なぜこの報告を行うか]

## 2. 調査方法 / 分析手法
[どのように情報を収集・分析したか]

## 3. 結果
### 3.1 [小見出し1]
### 3.2 [小見出し2]

## 4. 考察
[結果から言えること]

## 5. 結論・提言
[アクションにつながる提言]

## 6. 付録（必要に応じて）
[詳細データなど]
```

### 2.5.2 提案書テンプレート

```markdown
以下の内容で提案書を作成してください。

## 提案の概要
[何を提案するか]

## 提案先
[誰に提案するか]

## 背景情報
[関連する課題・状況]

## 出力形式

# [提案書タイトル]

## 1. 課題認識
[解決すべき課題を明確に]

## 2. 提案内容
### 2.1 提案の概要
### 2.2 具体的な施策
### 2.3 期待される効果

## 3. 実施計画
### 3.1 スケジュール
### 3.2 体制
### 3.3 必要リソース

## 4. 費用対効果
### 4.1 投資額
### 4.2 期待リターン
### 4.3 ROI試算

## 5. リスクと対策
[想定されるリスクと対応策]

## 6. 次のステップ
[承認後のアクション]
```

## 2.6 実践演習

### 演習1：情報のMECE整理

```
【課題】
以下の顧客フィードバックをMECEに整理してください。

「価格が高い」「UIが使いにくい」「サポートの返信が遅い」
「機能が足りない」「マニュアルがわかりにくい」
「動作が重い」「競合より高い」「デザインが古い」
「問い合わせ窓口がわかりにくい」「新機能が少ない」
```

### 演習2：ピラミッド構造での整理

```
【課題】
あなたは営業部門のマネージャーです。
以下のデータを基に、経営層への報告をピラミッド構造で整理してください。

データ：
- 今月の売上：前年同月比120%
- 新規顧客：15社獲得（目標10社）
- 既存顧客の解約：2社（前月は5社）
- 営業担当者の増員：2名
- 新商品の売上比率：全体の30%
```

### 演習3：SWOT分析

```
【課題】
架空の中堅IT企業「テックソリューションズ株式会社」のSWOT分析を行ってください。

企業概要：
- 設立20年、従業員200名
- 主力：業務システム開発
- 強み：大手企業との長期取引
- 課題：DX人材の不足
- 最近：AI事業への参入を検討中
```

### 演習4：提案書作成

```
【課題】
「社内会議の効率化」について、経営層への提案書を作成してください。

背景：
- 平均会議時間：90分/回
- 週あたり会議数：社員1人あたり平均8回
- 課題：資料準備に時間がかかる、議事録作成が負担
```

PREP法の正しい順序は？

- [ ] A) 理由 → 結論 → 例 → まとめ
