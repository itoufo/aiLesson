## 4.1 業務自動化の基礎

### 4.1.1 業務自動化とは

![業務自動化の効果](/docs/images/02_応用/04_automation_before_after.jpeg)

### 4.1.2 自動化に適した業務の特徴

| 特徴 | 説明 | 例 |
|------|------|-----|
| **定型的** | ルールが明確で繰り返し発生 | データ入力、ファイル整理 |
| **大量** | 処理件数が多い | 請求書処理、メール振り分け |
| **時間依存** | 特定時刻に実行が必要 | 日次レポート、リマインダー |
| **複数システム連携** | 異なるツール間のデータ移動 | CRM→スプレッドシート |
| **エラーが起きやすい** | 手作業だとミスしやすい | 転記作業、計算 |

### 4.1.3 自動化の段階

```
レベル1: 単純自動化
  └─ 1つのタスクを自動実行
     例：毎朝9時にメール送信

レベル2: 条件付き自動化
  └─ 条件分岐を含む自動実行
     例：金額が10万円以上なら上長に通知

レベル3: 連携自動化
  └─ 複数ツール間の連携
     例：フォーム入力→DB登録→Slack通知→カレンダー登録

レベル4: AI連携自動化
  └─ AIによる判断を含む自動化
     例：メール内容をAIが分析→適切な担当者に自動振り分け
```

## 4.2 Notionの活用

### 4.2.1 NotionとNotion AI

**Notion**は、ドキュメント、データベース、タスク管理を統合したワークスペースツールです。

![Notionの主な機能](/docs/images/02_応用/04_notion_features.jpeg)

### 4.2.2 Notion AIの主な機能

| 機能 | 説明 | 使い方 |
|------|------|--------|
| **文章生成** | 新しい文章を作成 | 「〇〇について書いて」 |
| **要約** | 長文を短くまとめる | テキスト選択→「要約」 |
| **翻訳** | 多言語翻訳 | テキスト選択→「翻訳」 |
| **トーン変更** | 文体の調整 | 「フォーマルに」「カジュアルに」 |
| **続きを書く** | 文章の続きを生成 | カーソル位置で「続きを書く」 |
| **アクションアイテム抽出** | タスクを自動抽出 | 議事録から「アクションアイテム抽出」 |

### 4.2.3 Notion AI活用プロンプト集

```markdown
【議事録からアクションアイテム抽出】
この議事録から、以下の形式でアクションアイテムを抽出してください：
- タスク内容
- 担当者
- 期限
- 関連する議題

---

【ブレインストーミング】
「[テーマ]」について、以下の観点からアイデアを10個出してください：
- 実現可能性が高いもの
- 斬新なもの
- コストを抑えられるもの

【テンプレート作成】
[業務名]のテンプレートを作成してください。
含める項目：
- 基本情報
- 目的
- 手順

---

【FAQ作成】
[製品・サービス名]についてのFAQを作成してください。
- 初心者向けの質問を5つ
```

## 4.3 Zapierの活用

### 4.3.1 Zapierとは

**Zapier**は、異なるWebアプリケーション間を連携させる自動化プラットフォームです。

![Zapierの基本概念](/docs/images/02_応用/04_zapier_concept.jpeg)

### 4.3.2 Zapierの基本用語

| 用語 | 説明 |
|------|------|
| **Zap** | 自動化ワークフロー全体 |
| **Trigger** | Zapを起動するきっかけ |
| **Action** | Triggerに応じて実行される処理 |
| **Filter** | 条件に合う場合のみ処理を続行 |
| **Path** | 条件分岐で異なる処理を実行 |
| **Formatter** | データの形式を変換 |

### 4.3.3 よく使われるZapの例

```
【例1：問い合わせ管理】
Trigger: Googleフォームに回答が送信される
  ↓
Action 1: Google Sheetsに記録
  ↓
Action 2: Slackの#問い合わせチャンネルに通知
  ↓
Action 3: 担当者にメール送信

---

【例2：SNS投稿の効率化】
Trigger: NotionのDBに新しい記事が追加される
  ↓
Filter: ステータスが「公開」の場合のみ
  ↓
Action 1: Twitterに投稿
  ↓
Action 2: Facebookに投稿
  ↓
Action 3: LinkedInに投稿

---

【例3：経費精算の自動化】
Trigger: 特定フォルダにレシート画像がアップされる
  ↓
Action 1: OCRで文字を抽出（AI連携）
  ↓
Action 2: Google Sheetsに経費データを記録
  ↓
Path: 金額によって分岐
  ├─ 1万円未満: 自動承認、記録のみ
  └─ 1万円以上: 上長にSlackで承認依頼
```

## 4.4 その他の自動化ツール

### 4.4.1 ツール比較

| ツール | 特徴 | 適した用途 | 価格帯 |
|--------|------|------------|--------|
| **Zapier** | 7000+アプリ連携、UI簡単 | 汎用的な自動化 | 無料〜 |
| **Make (旧Integromat)** | 複雑なロジック、コスパ良 | 高度な自動化 | 無料〜 |
| **Power Automate** | Microsoft製品との親和性 | Office365環境 | Microsoft 365に含む |
| **n8n** | オープンソース、セルフホスト可 | カスタマイズ重視 | 無料（セルフホスト） |
| **IFTTT** | シンプル、個人向け | 簡単な連携 | 無料〜 |

### 4.4.2 Microsoft Power Automate

![Power Automateの特徴](/docs/images/02_応用/04_power_automate.jpeg)

### 4.4.3 選定の考え方

```markdown
【ツール選定フローチャート】

Q1: 主にMicrosoft製品を使っていますか？
  ├─ Yes → Power Automate
  └─ No → Q2へ

Q2: 複雑な条件分岐やデータ加工が必要ですか？
  ├─ Yes → Make (Integromat)
  └─ No → Q3へ

Q3: 自社サーバーで運用したいですか？
  ├─ Yes → n8n
  └─ No → Zapier

Q4: 個人利用で簡単な自動化のみ？
  └─ Yes → IFTTT
```

## 4.6 実践演習

### 演習1：Notion AIでの業務効率化

```
【課題】
週次ミーティングの議事録テンプレートを作成し、

- 要約の自動生成
```

### 演習2：Zapierフロー設計

```
【課題】
以下の業務を自動化するZapを設計してください。

現在の業務：
1. 顧客からの問い合わせフォーム（Googleフォーム）を確認
2. 内容をスプレッドシートに手動で転記
3. 担当者にメールで連絡
4. Slackの#問い合わせチャンネルにも投稿
5. 対応が完了したらフォームのステータスを更新

課題：
- すべて手作業で1件あたり15分かかっている
- 転記ミスが月に2-3件発生
```

### 演習3：自動化ROI計算

```
【課題】
以下の自動化案のROIを計算してください。

自動化対象：月次レポート作成
現状：
- 作業時間：月20時間
- 担当者時給：3,000円
- 作成ミス率：5%（ミス1件あたり修正に2時間）

自動化後：
- 作業時間：月2時間（確認作業のみ）
- ツール費用：月5,000円
- 構築工数：40時間（一回のみ）
- 保守工数：月2時間
```
